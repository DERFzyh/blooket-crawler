🔬 核心技术原理

React Fiber 状态遍历

Blooket 使用 React 构建。React 在每个 DOM 元素上存储了一个 __reactFiber$* 属性， 指向对应的 fiber 节点。通过遍历 fiber 树，可以访问任何组件的内部 state 和 props，无需逆向工程或 API 破解。 

// 遍历 React fiber 树查找游戏状态
for (const el of document.querySelectorAll('*')) {
  const key = Object.keys(el).find(k => k.startsWith('__reactFiber'));
  if (!key) continue;
  let node = el[key];
  for (let i = 0; i < 40 && node; i++) {
    const s = node.stateNode?.state;
    // s.question.correctAnswers  ← 正确答案
    // s.gold, s.weight, s.crypto ← 游戏数据
    node = node.return || node.child || node.sibling;
  }
}

自动答题流程

  1. 每 400ms 遍历页面所有元素的 React fiber 树
  2. 找到包含 question.correctAnswers 的组件状态
  3. 在 DOM 中搜索 class*="answerContainer" 元素
  4. 匹配 textContent 与正确答案
  5. 自动点击匹配的元素
  6. 在 stage === 'feedback' 时自动点击 Next 按钮

游戏作弊原理

游戏作弊方式修改的状态字段
🎣 Fishing Frenzy 直接修改 React state 并调用 forceUpdate() state.isFrenzy=true, state.weight=N, state.lure=N

🧪 已知限制

  • Cookie 弹窗 — Blooket 每次页面加载都弹出 cookie 同意框，需延迟等待自动关闭
  • iframe 隔离 — 部分游戏内容在 iframe 中，需通过 page.frames() 访问
  • 多玩家要求 — Fishing Frenzy 等游戏需要 ≥2 名玩家才能开始
  • NextJS RSC — PIN 输入需要通过真实键盘打字触发 React onChange，JS 赋值无效
  • 游戏创建 — 全自动创建游戏需要处理 School Identification 弹窗，当前半自动化
